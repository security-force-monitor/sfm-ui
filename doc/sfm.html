When implementing the frontend, here are some changes to the design to keep in mind:

All pages::

If bold text appears under an organization’s name, it’s that organization’s “root_name” (i.e. “Police” or “Military”), not it’s parent or anything else. The page designs are inconsistent in this respect; ignore the inconsistencies.

P6 Event + Map:

Ignore the confidence and sources for nearby organization. That info doesn’t exist.

P7 Organization + Map:

Affiliated Persons: Ignore the place of birth. That info doesn't exist.

P8 Organization Dossier:

  • Add Affiliated Persons
  • Sites and areas should be listed for all time. Only the current location map deals with the present.
  • Sites (Headquarters) have an admin level 1 and 2; they don’t have a city and state. They do have a name.
  • Ignore the bold parent in Other Affiliations. No such thing.
  • Ignore the location description for superior, subordinate and other affiliations. The dossier shows data for all time. It only makes sense to show a location description that is relevant to the period in which the organization was related to the other organization, but there may be many such location descriptions, and it will just be crazy to list all those details.

P7 and P8:

  • Ignore the confidence for alleged events. That info doesn’t exist.

P9 Organization + Chart:

  • This page should have confidences and sources just like P7. Better to ignore this sidebar and replicate P7’s sidebar.

P10 Person + Chart:

  • Remove the place of birth and death date.
